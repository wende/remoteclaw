# Tool Catalog Generation

> **NOTE**: Static catalog is now the **fallback**. Primary discovery is dynamic — see `dynamic-tool-discovery.md`. The static catalog is used when the gateway runs without access to the OpenClaw dev source (e.g., production deployments without the source tree).

## Why Static Catalog

`createOpenClawTools()` is an internal function in the OpenClaw codebase — it's not exported from any public API (`openclaw/plugin-sdk` or the main package entry). The installed npm package bundles everything into dist files with hashed names, making runtime import impossible from a plugin.

Solution: generate a static JSON catalog at build time from the dev source, load it at runtime.

**Limitation**: The static catalog only contains built-in tools — it cannot include plugin-registered tools since those only exist at runtime. This is why dynamic discovery was added.

## How It Works

`scripts/generate-catalog.cjs` uses `jiti` to load TypeScript source directly:

```
jiti (local) → ~/projects/openclaw/src/agents/openclaw-tools.ts
                  ↓ imports @sinclair/typebox, etc.
                  ↓ resolved from symlinked node_modules
              → createOpenClawTools({})
              → 16 tools with name, description, JSON Schema parameters
              → writes src/tool-catalog.json
```

## Prerequisites

1. **OpenClaw dev source** at `~/projects/openclaw` (or pass path as arg)
2. **Symlinked node_modules**: The dev source needs a `node_modules/` for package resolution. Since the dev tree doesn't have its own, we symlink from the installed package:
   ```
   ln -s /path/to/installed/openclaw/node_modules ~/projects/openclaw/node_modules
   ```
3. **jiti** installed locally in remoteclaw (devDependency)

## Running

```bash
npm run refresh-catalog
# or
node scripts/generate-catalog.cjs [openclaw-source-root]
```

## Approaches That Failed

| Approach | Why It Failed |
|----------|--------------|
| ESM via `tsx` | `NODE_PATH` not respected in ESM; packages resolve relative to importing file |
| Node `--experimental-strip-types` | Can't handle `.js` → `.ts` extension mapping (OpenClaw convention) |
| `tsx` with OpenClaw source | `__name is not a function` — tsx/esbuild conflicts with source patterns |
| jiti from installed openclaw's node_modules | `Cannot read properties of undefined (reading 'createRequire')` — bundled jiti has context issues |

## Output

`src/tool-catalog.json` — array of `{ name, description, parameters }` objects. The `parameters` field contains full JSON Schema generated by `@sinclair/typebox` at catalog generation time.

Current tools (16): browser, canvas, nodes, cron, message, tts, gateway, agents_list, sessions_list, sessions_history, sessions_send, sessions_spawn, subagents, session_status, web_search, web_fetch.
